
Time complexity:
  Breadth search і Depth search мають однакову складність O(n). Кожна нода буде відвідана 1 раз

Space complexity
  Breadth search використовує queue, і зберігає всі елементи одного рівня, на великому дереві, яке рівномірне
  че може бути багато елементів

  Depth search натомість використовує рекурсію і не зберігає дані у queue, але для глибоких дерев можна отримати
  переповнення стеку, особливо коли у нас дерево буде схоже на linked list
    1
     \
      2
       \
         3
          \
            4         
  Це найгірший випадок для Depth search

  Для binary search tree Depth Search InOrder повертає відсортовану послідовність!
